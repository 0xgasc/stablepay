RECOMMENDED

Method A: Embed Widget

1

Add the StablePay script to your site

Paste this line before the closing </body> tag on any page where you want payments:

<script src="https://stablepay-nine.vercel.app/js/stablepay-widget.js"></script>

For Lovable: Go to your page editor and add a Custom HTML/Script block

2

Add a Pay Button (Fixed Amount)

For a simple fixed-price product or subscription:

<button onclick="StablePay.checkout({
  merchantId: 'cmhkjckgi0000qut5wxmtsw1f',
  amount: 29.99,
  productName: 'Premium Subscription',
  chain: 'BASE_MAINNET',
  onSuccess: function(data) {
    alert('Payment successful! Order: ' + data.orderId);
    // Redirect to thank you page, unlock content, etc.
  },
  onError: function(err) {
    alert('Payment failed: ' + err.message);
  }
})">
  Pay $29.99 with USDC
</button>
2b

For carts or user-entered amounts:

<!-- Amount input -->
<input type="number" id="payAmount" placeholder="Enter amount" min="0.01" step="0.01" />

<button onclick="StablePay.checkout({

  Pay with USDC

Checkout Options Reference

All parameters you can pass to StablePay.checkout()

Parameter Type Required Description
merchantId string YES Your merchant ID
amount number YES Payment amount in USD (e.g. 29.99)
productName string optional Product or service name shown in checkout
customerEmail string optional Customer's email (for receipts)
customerName string optional Customer's name
chain string optional Blockchain to pay on (see below). If omitted, shows chain selector
metadata object optional Custom data (plan IDs, user IDs, etc.)
onSuccess function optional Called after payment. Receives { orderId, status }
onCancel function optional Called when user closes checkout
onError function optional Called on error. Receives { message }

FAQ

Do I need a backend?

No. The widget handles everything client-side. Just add the script and button.

What wallets do customers need?

Any EVM wallet: MetaMask, Coinbase Wallet, Rainbow, Trust Wallet, or any WalletConnect-compatible wallet.

When do I receive the funds?

Immediately. USDC is sent directly to your wallet. There's no holding period.

What are the fees?

A small platform fee (0.3% - 1% depending on plan) is invoiced separately. You receive 100% of customer payments.

How do refunds work?

Request a refund from your merchant dashboard. The admin reviews, approves, and processes refunds manually via on-chain transfer.

Can I test without real money?

Yes! Use chain: 'BASE_SEPOLIA' to test with testnet USDC. No real funds needed.
